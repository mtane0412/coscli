# セキュリティポリシー

coscli (バイナリ名 `cos`) のセキュリティに関する報告窓口・サポート方針・脅威モデルを記載します。

## サポートバージョン

| バージョン | サポート状況 |
|---|---|
| 最新リリース (現行 v0.2.x) | ✅ サポート対象 |
| `main` ブランチ (開発中スナップショット) | ✅ サポート対象 |
| それ以前のバージョン | ❌ サポート対象外 |

## 脆弱性の報告方法

本プロジェクトは個人開発の CLI ツールです。報告内容の重大度に応じて窓口を使い分けてください。

### 軽微な脆弱性 (誤設定・情報漏洩リスクが低いもの)

通常の [GitHub Issue](https://github.com/mtane0412/coscli/issues/new) として報告してください。

### 重大な脆弱性 (セッション窃取・機密情報漏洩・認証バイパスなど)

修正前に公開されることを避けるため、以下のメールアドレスに直接ご連絡ください。

**mtane0412@gmail.com**

件名に `[coscli security]` プレフィクスを付けていただけると対応しやすくなります。

---

報告いただく際は、以下の情報を含めていただけると助かります。

- 影響を受けるバージョンまたはコミット
- 再現手順 (最小ケース)
- 影響範囲 (情報漏洩 / 整合性破壊 / DoS など)
- 想定される攻撃者像
- 任意: 推奨される修正方針

## 対応プロセス

個人開発プロジェクトのため、厳密な SLA は提供しません。

- 受領確認は可能な限り **7 日以内**を目指します
- 修正の優先度は重大度・影響範囲に応じて判断します
- 修正完了後、報告者をクレジットします (希望に応じて匿名対応も可能です)

## 脅威モデル

本ツールは以下のアクターからの脅威を主な対象としています。詳細は [docs/security-audit-2026-05.md](docs/security-audit-2026-05.md) を参照してください。

| アクター | 想定シナリオ |
|---|---|
| 一般ユーザー | CLI を手動実行。誤操作・誤設定による機密情報の意図しない公開 |
| AI エージェント | プロンプトインジェクションで `--from-file` 等の引数を操作され、機密ファイルを Cosense に書き込む |
| ローカル攻撃者 | 同一マシン内の別プロセスから OS キーチェーンや設定ファイルの認証情報を読み取る |
| ネットワーク攻撃者 | `cos serve` の Bearer トークン推測、HTTP redirect 経由での Cookie 漏洩を試みる |

## 適用範囲外

以下は本ポリシーの対象外です。

- 利用者の環境設定起因の問題 (OS キーチェーン未セットアップなど)
- README や docs に既知の制限事項として明記されている挙動
- 依存ライブラリの脆弱性 (該当 upstream に報告してください)
